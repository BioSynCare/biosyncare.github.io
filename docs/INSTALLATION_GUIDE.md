## Installation FAQ

### Q: Do I need to download anything?
**A:** No! It installs directly from the website. No app store required.

### Q: How much space does it take?
**A:** Very small! Approximately 5-10 MB including all cached assets.

### Q: Does it work offline?
**A:** Yes! Once installed, the app works completely offline.

### Q: Can I uninstall it?
**A:** Yes, easily:
- **Desktop**: Right-click icon â†’ "Uninstall" or remove from Apps & Features
- **Mobile**: Long-press icon â†’ "Remove" or "Uninstall"

### Q: Will it update automatically?
**A:** Yes! Updates download in the background. You'll see an "Update Available" notification with a button to activate the update.

### Q: Can I use it without installing?
**A:** Absolutely! The web version works perfectly. Installation is optional but provides offline access and a better experience.

### Q: Is my data safe?
**A:** Yes! All data stays on your device. Nothing is sent to external servers unless you explicitly enable cloud sync.

### Q: Does it drain battery?
**A:** No. It uses the same resources as the web version. Safety limits prevent excessive usage.

### Q: Can I install on multiple devices?
**A:** Yes! Install on as many devices as you want.

## Troubleshooting

### Install button doesn't appear

**Possible causes:**
1. Browser doesn't support PWAs (use Chrome/Edge)
2. App already installed (check if icon exists)
3. HTTPS not enabled (only works on localhost or HTTPS sites)

**Solution:**
- Use Chrome, Edge, or Safari
- Check if app is already installed
- Try the browser menu method instead

### App won't install

**Check:**
1. Browser is up to date
2. Enough disk space available
3. Not in incognito/private mode

**Solution:**
- Update browser to latest version
- Free up disk space
- Use normal browsing mode

### Install icon disappeared after dismissing

**Why:** Chrome only shows the prompt once per session after dismissal.

**Solution:**
- Hard refresh page: `Ctrl+Shift+R` (Windows/Linux) or `Cmd+Shift+R` (Mac)
- Or clear browser cache
- Or use the on-page "Install App" button

### Update notification keeps appearing

**Why:** Service worker has an update but wasn't activated.

**Solution:**
- Click "Update Now" button
- Or close all instances of the app and reopen
- Or: DevTools â†’ Application â†’ Service Workers â†’ "Update on reload"

## Advanced: Programmatic Installation

Developers can trigger installation programmatically:

```javascript
// Import the installer
import { pwaInstaller } from './src/core/pwa-installer.js';

// Check if installable
if (pwaInstaller.canInstall()) {
  // Trigger install prompt
  const accepted = await pwaInstaller.promptInstall();
  console.log(accepted ? 'User installed!' : 'User dismissed');
}

// Check if already installed
if (pwaInstaller.isInstalled()) {
  console.log('Running in standalone mode');
}

// Get full status
const status = pwaInstaller.getStatus();
console.log(status);
// {
//   canInstall: boolean,
//   isInstalled: boolean,
//   isOnline: boolean,
//   updateAvailable: boolean,
//   swRegistered: boolean
// }
```

## Platform-Specific Tips

### iOS Tips
- Use Safari for installation (Chrome on iOS doesn't support PWA install)
- Once installed, it runs in standalone mode (no Safari UI)
- Push notifications require explicit permission

### Android Tips
- Chrome, Edge, and Samsung Internet all support PWA install
- Adaptive icons work on Android 8.0+
- App shortcuts appear in long-press menu

### Windows Tips
- Pin to Start Menu or Taskbar for quick access
- Runs as native window (separate from browser)
- Can run on startup via Windows settings

### macOS Tips
- Safari's "Add to Dock" is the native method
- Chrome/Edge method works but less integrated
- App appears in Applications folder

### Linux Tips
- Works best with Chrome/Chromium
- Desktop file created in `~/.local/share/applications/`
- Integrates with any desktop environment (GNOME, KDE, XFCE, etc.)
