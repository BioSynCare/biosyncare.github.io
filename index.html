Audiovisual Medicine

Protocolos de estimulação neurosensorial acessíveis para serem escutados e vistos em tempo real, diretamente no navegador. 100% client‑side.

Saiba mais Site atual Código (GitHub)

Demonstração visual/sonora aparece aqui quando o laboratório estiver ativo.

O que é

Audiovisual Medicine é um conjunto de protocolos com padrões sonoros e visuais (p.ex., senóides puras, batidas binaurais/monourais, isócronos, simetrias, variações cromáticas, ritmos respiratórios) executados em tempo real no navegador para neuromodulação não invasiva e de baixo custo. O modelo é JS apenas no cliente, hospedado em GitHub Pages.

Os princípios seguem o artigo de referência da casa e a tradição consolidada de AVBE; as técnicas base já estão implementadas e reutilizadas dos repositórios aeterni, harmonicare e audiovisualmedicine.

Cliente‑side, por quê?

  • Baixa latência e máxima fidelidade (DSP local).
  • Privacidade por padrão (sem áudio subir ao servidor).
  • Facilidade de distribuição (Pages / PWA).

Integração com a ONC (Open Neurosensory Commons)

A ONC fornece vocabulários/ontologias (OWL+SKOS) e JSON‑LD para descrever protocolos, técnicas, parâmetros, evidências e organizações. Abaixo, um parecer pragmático para o Audiovisual Medicine:

Quando usar

  • Para documentar cada protocolo (parâmetros, indicações, segurança) em JSON‑LD/OWL.
  • Para reuso científico (links para fontes, versões, evidências).
  • Para governança e interoperabilidade (SPARQL, Wikidata/OpenAlex).

Quando não usar

  • Em protótipos ultra‑rápidos onde o schema atrasa a entrega.
  • Para features puramente runtime (DSP) que não exigem metadados semânticos.
  • Se o overhead de serialização impactar a UX.

Benefícios no espectro ONC

  • Escrita: reduz ambiguidade, define nomes e faixas válidas.
  • Desenvolvimento: valida presets, automatiza testes de parâmetros.
  • Manutenção: versionamento semântico de protocolos e evidências.
Decisão recomendada (MVP → ONC‑ready)
  1. MVP lean com presets locais (JSON) e tipos TS/JS bem definidos.
  2. Adapter ONC: função pura que mapeia presets ⇄ JSON‑LD/OWL.
  3. Export opcional: botão “Exportar para ONC” para cada protocolo.

Engine de Áudio & Reuso

Reutilize os módulos já disponíveis nos repositórios aeterni, audiovisualmedicine e harmonicare (histórico). O núcleo inclui: geradores senoidais, batidas binaurais/monourais, isócronos, envelopes de amplitude, LFOs e sincronização respiratória.

  • API proposta: engine.play({ type: 'binaural', base: 300, beat: 8, duration: 30 })
  • Sincronização: engine.breathe({ pattern: '4-7-8' })
  • Visual: viz.flicker({ freq: 8, waveform: 'square' })

Obs.: Nesta página, os módulos são carregados on‑demand via type="module". Nada de servidor.

Use fones de ouvido. Controle o volume do sistema.

Como organizar o código (client‑only)

src/
  engine/
    audio.js        // Web Audio primitives (osc, gain, envelope)
    binaural.js     // helpers para batidas binaurais/monourais
    breathe.js      // sincronização respiratória
  viz/
    flicker.js      // padrões visuais (canvas/webgl)
  presets/
    relax.json      // parâmetros nomeados
    foco.json
  onc/
    adapter.js      // mapeia presets ⇄ JSON‑LD (opcional)
index.html          // esta página
assets/             // favicon, og, logos

FAQ

Precisa de servidor?

Não. Tudo roda no cliente (Web Audio + Canvas/WebGL). Integrações ONC e exportações são arquivos locais/baixados.

Compatível com mobile?

Sim, Safari/Chrome mobile modernos. Recomendado usar fones de ouvido.

Como versionar protocolos?

Via JSON local (SemVer) e, quando habilitado, exportar JSON‑LD ONC com version IRI.

Boas práticas

  • Ganho seguro (limitar a −6 dBFS) e fade suave.
  • Evitar flicker visual acima do limiar desconfortável do usuário.
  • Documentar contra‑indicações e uso responsável.
